Remediation Steps
① Rotate immediately
Revoke and regenerate every exposed secret from the provider's dashboard before doing anything else. Assume the secret was already found.
② Purge git history
Use git filter-repo to remove the secret from all commits, then force-push. Every team member must re-clone.
③ Use environment variables
Store secrets in .env files (gitignored), CI/CD secret stores, or a vault (HashiCorp Vault, AWS Secrets Manager, Azure Key Vault).
④ Enable push protection
Turn on GitHub Secret Scanning + Push Protection in Settings → Code Security → Secret Scanning to block future leaks at push time.
⑤ Audit access logs
Check provider audit logs for any unauthorised API calls made with the exposed credential. Document findings for your incident record.
